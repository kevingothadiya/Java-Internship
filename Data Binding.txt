Data Binding
  - Data Binding is a technique where the data stays in sync between the component and the view
  - Whenever user updates the data in the view angular updates the component and when the component gets new data the angular update the view
  - Two way to bind data in Angular:

    a. One Way Binding
      1. Component to view
        i.  Interpolation
        ii. Property Binding
        iii. Class Binding
        iv. Style Binding

      2. View to Component
        i. Event Binding

    b. Two way Binding
      1. ngModel

// One Way Binding:

# Property Binding:
  - HTML Attribute Vs DOM Property 
    HTM Attribute:  value of an attribute is constant

  - Property binding is a one way binding technique, which will help to bind values to the properties of HTML elements.
  - Whenever the value of the component changes, the angular updates the element property in the view
  - We can set the properties such as class, href, src etc. using property binding.
  - Sytax:

    [binding-target]="binding-source"
  - Example:
    <h1 [disabled]="isDisabled"> I am Disabled </h1>
    in ts: isDisabled = true


# class Binding:
  - The angular class binding is used to add or remove classes to and from the HTML elements.
  - We can add CSS classes conditionally to an element.
  - The Angular provides three ways to add/remove classes to and from the element:

  a. Using the DOM 'className' property
  b. Using the 'class' property
  c. Using the 'NgClass' directive

  -> Class binding with className
    <h2 [className]="'myColor, myFontStyle'"> Hello </h2>

  -> Class binding with class
    <h2 [class]="'myColor, myFontStyle'">Hello</h2>
    <h2 [class.myColor]="condition">Hello</h2>
    <h2 [class] = "MyCall()">Hello</h2>
    <h2 [class] = "condition?'myColor':'myFontStyle'"> Hello </h2>

  -> Class binding with NgClass
    - It offers more flexibility like easily adding multiple classes etc.
    - Using ngClass we can create dynamic styles in angular components by using conditional expressions.

    - ngClass with String:
      <h2 [ngClass]="'myColor myFontStyle'">Hello</h2>
    - ngClass with Array:
      <h2 [ngClass]="['myColor','myFontStyle']">Hello</h2>
    - ngClass with Object:
      <h2 [ngClass]="{'myColor':true}">Hello</h2>

    - ngClass without square bracket:
      <h2 ngClass='myColor'>Hello </h2>

#Service in Angular:

  => Without using Service: Duplication of Data

  1. What is an Angular Service? : 

    Service is a piece of reusable code with a focused purpose. A code that we will use across multiple components in our application.

    Our components need to access the data. we can write data access code in each Component, but this is very inefficient and breaks the rule of single responsibility (DRY). The Component should focus on presenting the data to the user.

    The task of receiving data from the back-end server should be delegated to another class. We call a class - a service class because it provides each Component with the data it needs.

    "Angular services provide a way for us to separate Angular app data and functions that can be used by multiple components in our app. To be used by multiple components, a service must be made injectable. Services that are injectable and used by a component become dependencies of that component. The component depends on those services and can't function without them."

  => Service is just a class with specific purpose:
    a. Data sharing across multiple components
    b. It is used to apply application logic
    c. We can use Services for external interactions such as connecting to a APIs,database.

  2. Dependency injection? :

    Dependency injection is the mechanism that manages the dependencies of an app's components and the services that other components can use.

  3. Create a new service :

    ng generate service api-access --skip-tests

  4. Registration of Angular Service

    import { DataService } from './data.service'
    Navigate to our app module file -> register the service under providers: [DataService],

  4.  Inject the new service into Component

  1. path:'**' with component 
  2. path:'**' with redirectTo:'/home' pathMatch=:'full' / 'prefix' (Default)

- pathMatch: 'prefix' (Default behavior)
  a.  - Angular checks whether the URL starts with the given path.
    - If it starts with that path → route matches.

      const routes: Routes = [
        { path: '', component: HomeComponent, pathMatch: 'prefix' }
      ];

    - Now these URLs will match:
         /
       /about
       /contact
       /dashboard
    - Because all URLs start with empty string ('').

# JWT Authentication in Angular :
  Login → store token in localStorage
  Interceptor → attach token automatically to API requests
  AuthGuard → protect routes
  Logout → clear token

- Architecture flow : Login → Save token → Interceptor adds token → Guard checks token → Access secured routes

#Step -1: Store token after Login

  - When user logs in successfully, backend returns JWT token.
    {
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }


  - this.authService.login(credentials).subscribe(res => {

#Step-2: Create Auth Service
  export class AuthService {
      constructor() {}

      saveToken(token: string) {
      localStorage.setItem('token', token);
      }

      getToken(): string | null {
      return localStorage.getItem('token');
      }

      isLoggedIn(): boolean {
      return !!localStorage.getItem('token');
      }

      logout() {
      localStorage.removeItem('token');
      }
    }

#Step-3: HTTP Interceptor (Attach Token Automatically) : Attach JWT token in every API request header

    - Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..

    -  export const authInterceptorInterceptor: HttpInterceptorFn = (req, next) => {

        const storageService =inject(MyLocalStorage)
        const skipUrls=['/login','/registration']
        const shouldSkipUrl=skipUrls.some(u=>req.url.includes(u));
        if(shouldSkipUrl)
        {
        return next(req)
        }

        const token=storageService.getItem('token');
        if(token)
        {
        const cloneReq=req.clone({
          setHeaders:{
          Autherization:'Bearer '+token,
          'Content-Type': 'application/json'
          }
        })
        return next(cloneReq)
        }
        return next(req);
      };

#Step -5 : AuthGuard (Protect Routes)

    - Prevent unauthorized users from opening secure pages

    - export class AuthGuard implements CanActivate {
      constructor(
      private authService: AuthService,
      private router: Router
      ) {}

      canActivate(): boolean {

      if (this.authService.isLoggedIn()) {
        return true;
      }

      this.router.navigate(['/login']);
      return false;
      }
    }

    or

    export const authGuardGuard: CanActivateFn = (route, state) => {

      const myStorageSerivce=inject(StorageService)
      const myRouter=inject(Router)
      const token=myStorageSerivce.getItem('authToken')

      if(token)
      {
      return true;
      }
      myRouter.navigate(['/login'])
      return false;
    };

<strong></strong>#Local Storage:
  - In Angular localStorage is used to store data in the browser permanently (until manually cleared)
  - It's commonly used for things like JWT tokens, user preferences, theme settings and cached data.

  - localStorage is a browser Web Storage API that stores data as key-value pairs (string format)
    and persists even after page refresh or browser restart
  - Example: localStorage.setItem('username','Deepak')

# Methods in localStorage API

  - Save data:
    localStorage.setItem('token','Bearer ssbhsjb78yuhfjuih');

  - Get data:
    localStorage.getItem('token');

  - Remove data:
    localStorage.removeItem('token');

  - Clear Everything:
    localStorage.clear();

  - Store Object:
    const userData= {
      name:'Deepak Vishwakarma',
      role: 'Admin'
    };

    localStorage.saveItem('user',JSON.stringify(userData));

  - Retrive Object:
    const userData=JSON.parse(localStorage.getItem('user') || '{}');




# Best practice
  - Instead of using localStorage directly everywhere, create a Storage Service:

  export class StorageService {	
      setItem(key: string, value: string) {
      localStorage.setItem(key, value);
      }

      getItem(key: string) {
      return localStorage.getItem(key);
      }

      removeItem(key: string) {
      localStorage.removeItem(key);
      }
    }

  - Use it like: this.storageService.setItem('token', token);
